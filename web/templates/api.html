API Documentation

Complete REST API for accessing status data, incidents, and real-time updates. All responses are JSON formatted.

Base URL
{{.BaseURL}}

Authentication

Public endpoints require no authentication. Admin endpoints (POST, PUT, DELETE) support multiple auth methods:

API Key Header

X-API-Key: your-key

Bearer Token

Authorization: Bearer token

Basic Auth

Authorization: Basic base64

Query Parameter

?api_key=your-key

Summary

GET /api/summary Cloudflare-style status summary

Response

{ "page": { "id": "status", "name": "System Status", "url": "{{.BaseURL}}" }, "status": { "indicator": "none", "description": "All Systems Operational" }, "components": [...], "incidents": [...] }

Status

GET /api/status All service statuses with history

Response

{ "success": true, "data": { "overall": "operational", "services": [ { "name": "API Server", "status": "operational", "uptime": 99.95, "response_time_ms": 145 } ] } }
GET /api/status/{name} Single service status

Example

GET /api/status/API%20Server

Components

GET /api/components List all monitored components

Response

[ { "id": "api-server", "name": "API Server", "status": "operational", "group": "Core Services", "uptime_percent": 99.95 } ]

Incidents

GET /api/incidents List all incidents

Query Parameters

active=true # Only active incidents limit=50 # Max results
POST /api/incidents Create new incident Auth Required

Request Body

{ "title": "Database Connection Issues", "message": "Investigating elevated error rates", "status": "investigating", "severity": "major", "affected_services": ["Database", "API"] }

cURL Example

curl -X POST {{.BaseURL}}/api/incidents \ -H "X-API-Key: your-key" \ -H "Content-Type: application/json" \ -d '{"title": "Issue", "status": "investigating", "severity": "minor"}'
PUT /api/incidents/{id} Update incident status Auth Required

Request Body

{ "status": "resolved", "message": "Issue has been resolved" }

History

GET /api/history 90-day uptime history

Query Parameters

days=90 # Number of days (default: 90)
GET /api/uptime Current uptime percentages

Response

{ "API Server": 99.95, "Database": 100.0 }

Metrics

GET /api/metrics System-wide metrics

Response

{ "total_services": 7, "operational_count": 7, "degraded_count": 0, "down_count": 0, "overall_uptime": 99.95, "average_response_ms": 187, "active_incidents": 0 }

WebSocket

WS /ws Real-time status updates

Connection

const ws = new WebSocket('ws://{{.Host}}/ws'); ws.onmessage = (event) => { const data = JSON.parse(event.data); console.log(data.type, data.service); };

Message Types

"initial" - Full status on connect "update" - Service status changed
